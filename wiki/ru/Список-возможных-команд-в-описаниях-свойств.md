## Синхронизация в пределах свойств одного объекта

`Sync_from` - читает значения ИЗ другого места и записывает В текущее свойство. `Sync_to` - записывает значение текущего свойства в другое место (свойство, параметр объекта, инфо о проекте).

| Команда                                                            | Описание                                                                    |
| ------------------------------------------------------------------ | --------------------------------------------------------------------------- |
| `Sync_from`                                                        | Читает значения из источника и записывает в текущее свойство                |
| `Sync_to`                                                          | Записывает значение текущего свойства в целевое место                       |
| **Чтение данных (`Sync_from`)**                                    |                                                                             |
| `Sync_from{ИМЯ_ПАРАМЕТРА}`                                         | Копирует значение параметра GDL в свойство (имя параметра на латинице)      |
| `Sync_from{ИМЯ_ПАРАМЕТРА; ИГНОРИРУЕМЫЕ_ЗНАЧЕНИЯ}`                  | Копирует значение параметра GDL с исключением указанных значений            |
| `Sync_from{description:ОПИСАНИЕ_ПАРАМЕТРА}`                        | Копирует значение по отображаемому имени параметра (поддерживает кириллицу) |
| `Sync_from{description:ОПИСАНИЕ_ПАРАМЕТРА; ИГНОРИРУЕМЫЕ_ЗНАЧЕНИЯ}` | Копирует значение по отображаемому имени с исключением значений             |

| `Sync_to{id}`                                                      | Записывает значение свойства в ID элемента (не работает в AC22)             |

## Игонрируемые значения (_v1.77+_)

- `empty` - Игнорировать пустые (для строк) и нулевые (для чисел)
- `trim_empty` - Игнорировать состоящие из пробелов значения (для строк) и нулевые (для чисел)
- `def` - сброс свойства к значению по умолчанию, если подходящие значения для записи не найдены

### Обработка массивов GDL объектов (только чтение) _v1.69+_

| Команда                          | Описание                                                                         |
| -------------------------------- | -------------------------------------------------------------------------------- |
| `Sync_from{ИМЯ_ПАРАМЕТРА; uniq}` | Возвращает уникальные значения массива (для текста) или сумму (для чисел)        |
| `Sync_from{ИМЯ_ПАРАМЕТРА; sum}`  | Возвращает конкатенированные значения массива (для текста) или сумму (для чисел) |
| `Sync_from{ИМЯ_ПАРАМЕТРА; max}`  | Возвращает максимальное значение массива                                         |
| `Sync_from{ИМЯ_ПАРАМЕТРА; min}`  | Возвращает минимальное значение массива                                          |

При необходимости указать диапазон чтения массивов - указывается сначала значения для строк, затем для столбцов, в формате `(ПЕРВАЯ_СТРОКА, ПОСЛЕДНЯЯ СТРОКА)(ПЕРВЫЙ СТОЛБЕЦ, ПОСЛЕДНИЙ СТОЛБЕЦ)`. Например, запись `Sync_from{ИМЯ_ПАРАМЕТРА; uniq(1,3)}` выведет уникальные значения, которые находятся с диапазоне от первой до третьей строки включительно. Также можно использовать GDL параметры объекта при задании диапазона, например `Sync_from{ИМЯ_ПАРАМЕТРА; uniq(1,nrow)}`

### Проверка наличия дробной части в координатах/угле

[Видео на youtube с примером](https://youtu.be/a9Dnvo1P-rk)
[Видео на rutube с примером](https://rutube.ru/video/c853151ae9a8b3582bd3a10fb52863f9)

Функции возвращает ИСТИНА, если дробная часть отсутствует и ЛОЖЬ, если дробная часть больше допуска.

Допуски - 0.001 мм для координат и длины, 0.00001 градуса для углов.

Отключить проверку координат можно двумя способами - отключив синхронизацию полностью (Установив значение свойства-флага ЛОЖЬ), либо создать булево свойство с описанием `Sync_correct_flag` и задать ЛОЖЬ ему. При этом отключится только сверка наличия дробной части для данного элемента, остальная синхронизация будет работать.

## Суммирование

[Файл с примером для АС25](https://github.com/kuvbur/AddOn_SomeStuff/blob/master/wiki/files/%D0%A1%D1%83%D0%BC%D0%BC%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%90%D0%A125.pln)

Суммируются значения свойств у элементов с одинаковым свойством-критерием.

Численные значения суммируются, текстовые - соединяются только уникальные значения. По умолчанию разделитель стоит точка с запятой.
Флаг суммирования не требуется, суммируются значения всех выделенных элементов. После суммирования автоматически запускается синхронизация выделенных элементов.

_v1.74_ Если выделен только один элемент - будут обработаны все элементы с видимых слоёв, у которых доступны правила суммирования выбранного элемента.

- `Sum{Property:ИМЯ_ГРУППЫ/ИМЯ_СУММИРУЕМОГО_СВОЙСТВА; Property:ИМЯ_ГРУППЫ/ИМЯ_СВОЙСТВА_КРИТЕРИЯ}`

- `Sum{Property:ИМЯ_ГРУППЫ/ИМЯ_СУММИРУЕМОГО_СВОЙСТВА; Property:ИМЯ_ГРУППЫ/ИМЯ_СВОЙСТВА_КРИТЕРИЯ; разделитель}` для задания разделителя, отличного от ;

- _v1.72_ `Sum{Property:ИМЯ_ГРУППЫ/ИМЯ_СУММИРУЕМОГО_СВОЙСТВА; Property:ИМЯ_ГРУППЫ/ИМЯ_СВОЙСТВА_КРИТЕРИЯ; max}` - вывод максимального значения, для числовых свойств

- _v1.72_ `Sum{Property:ИМЯ_ГРУППЫ/ИМЯ_СУММИРУЕМОГО_СВОЙСТВА; Property:ИМЯ_ГРУППЫ/ИМЯ_СВОЙСТВА_КРИТЕРИЯ; min}` - вывод минимального значения, для числовых свойств
